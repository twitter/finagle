CachePoolCluster

object CachePoolCluster

Cache specific cluster implementation. - A cache pool is a Cluster of cache nodes. - cache pool requires a underlying pool manager as the source of the cache nodes - the underlying pool manager encapsulates logic of monitoring the cache node changes and deciding when to update the cache pool cluster

  16. def newUnmanagedZkCluster(zkPath: String, zkClient: ZooKeeperClient): Cluster[CacheNode]

    Zookeeper based cache pool cluster.

    Zookeeper based cache pool cluster. The cluster will monitor the underlying serverset changes and report the detected underlying pool size. The cluster snapshot is unmanaged in a way that any serverset change will be immediately reflected.

    zkPath

    the zookeeper path representing the cache pool

    zkClient

    zookeeper client talking to the zookeeper, it will only be used to read zookeeper

  17. def newZkCluster(zkPath: String, zkClient: ZooKeeperClient, backupPool: Option[Set[CacheNode]] = None, statsReceiver: StatsReceiver = NullStatsReceiver): ZookeeperCachePoolCluster

    Zookeeper based cache pool cluster.

    Zookeeper based cache pool cluster. The cluster will monitor the underlying serverset changes and report the detected underlying pool size. The cluster snapshot will be updated during cache-team's managed operation, and the Future spool will be updated with corresponding changes

    zkPath

    the zookeeper path representing the cache pool

    zkClient

    zookeeper client talking to the zookeeper, it will only be used to read zookeeper

    backupPool

    Optional, the backup static pool to use in case of ZK failure. Empty pool means the same as no backup pool.

    statsReceiver

    Optional, the destination to report the stats to
